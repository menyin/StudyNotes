	您可以创建自定义助手，为其命名并添加工具（例如代码解释器）。
	适用于需要详细上下文管理和复杂工具的应用。
         Compare：
	Compare 是一个用于比较不同模型性能的工具。
	您可以输入不同模型的示例，然后比较它们的输出。
	适用于评估不同模型的优劣。
         Completions：
	Completions 是一个用于生成文本补全的 API。
	您可以输入提示，模型会生成完整的文本。
	适用于生成单个文本段落或补全句子。
	总之，Chat 和 Assistants 适用于构建对话式应用，而 Compare 和 Completions 则更侧重于生成文本。具体选择取决于您的应用需求和功能。

*prompt包含以下元素：
 Instruction：指令词，要执行的特定任务或指令。如解释、翻译、总结、生成、代码....
 Content：背景，包含外部信息或额外上下文信息，用于引导模型更好的相应。如我是一个小学生、你是一个面试官...
 Input Data：用户输入的内容或问题。如“将以下语句翻译为英文【在那桃花盛开的地方】”，这里的【在那桃花盛开的地方】就是Input Data。  使用技巧是用三引号或其它符号框起来以助于模型理解。
 Output Indicator：输出格式。如请以json格式输出，20个字以内...

*prompt技巧：
 *语义明确：如“为openai写一首诗”-->“为openai写一首五言绝句的古诗”
 *用分割符：“将下面语句翻译为英文"""用华丽的语句"""”，三引号括起来的内容是要翻译的内容。 当然也可以用其他分隔符
 *指定输出格式：明确告诉GPT最后结果要用什么格式输出，如“列出国内最流行的ai框架，并用逗号分割”
 *列出示例：可以给GPT举个例子，然后让GPT去学习理解，这有助于GPT理解用户输入内容和为用户输出结果。  注意：可以多举个例子让GPT去理解学习，特别是一些推理类型问题，可以多举个类比例子
 *角色扮演：“你将扮演一个面试官”，“你担任一个面试官”，“你担任一个面试官，用户是一个初入职场的小白求职者”   注意角色设定是包含系统和用户的

*Dify工作流
   节点直接的输入输出要注意类型。注意两个节点之间输入输出变量的类型要衔接对应
   注意：代码执行节点输出json对象中，如果name属性值有null，到下游的模板转换节点时接收json会出问题报错（即使模板里没有输出该属性也会报错）。要替换为'N/A'，如{{ item.metadata.score | default('N/A') }}
            代码执行节点输出json对象中，如果name属性值有\n，到下游的模板转换节点时接收json会出问题报错。要替换为'\n\n'，如{{ item.content | replace('\n', '\n\n') }}
            原因：工作流中知识库或者其它扩展API节点返回给下游的变量如果有json并且包含值为null的属性，则不管是“模板转换节点”或者“代码执行（python3）”节点在接收该变量时会出现类型无法转换错误
            解决办法：加一个“代码执行（JavaScript）”节点来对null值进行转换。因为知识库之类的节点输出的都是固定的Array[Object]对象，因此如果有null值无法给到“代码执行（python3）”节点处理。


function main(data) {
    var arg11=data.arg1;
    arg11[0].metadata.score=0;
    return {
        result: arg11
    }
}
